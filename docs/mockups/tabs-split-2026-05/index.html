tonton — ver.3 (C案: Banking Metaphor 全振り)

リサーチ知見を反映: dual-axis 重ね描きはプロアプリ誰もやってないので捨てる。 tonton の差別化軸「カロリー貯金 = ダイエット銀行」を UI の主役にして、 Deficitly / Happy Scale / MacroFactor の良いとこ取り。

ver.3 (C案) の設計思想:
 ・Hero = 銀行残高ゲージ: 円形プログレスリング + 累計貯金 + 今日の入出金（取引メタファ）
 ・体重チャート = kg軸単独（Happy Scale 流）: 実測折線 + 理想ペース破線 + 目標水平線。kcalは混ぜない
 ・日別収支スパークライン: 棒グラフを小さく置く。+ / − / 純貯金の3KPIだけ添える
 ・月次目標達成度カードは ver.2 と同じ（目標/理論/実測 3セル）
 ・ご褒美画面は LLM 提案版（変更なし）

C案の特徴: Hero を「銀行残高ゲージ」化。円形プログレスは月目標までの達成率を表現、下に取引（入金/出金/平均）を3列で表示して家計簿アプリの語彙を借用。
