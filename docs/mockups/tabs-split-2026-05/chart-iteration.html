App Store 安全側の設計原則:
・LLM は パターン分類だけ担当 (free-form generation はしない)
・解説本文・「一般的に〜」「〜と言われている」「〜が知られている」テキストは事前審査済みテンプレ。LLM は数字穴埋めのみ
・命令形/指示形を避け、「振り返りメモ」として疑問形で問いかける
・タイトルは 📚 パターン辞典マッチング (医療AI ではなく辞書ツール)
・各カード末尾にディスクレーマー: ※ 一般情報の提示です。医療判断は専門家にご相談ください。
・「アクション/対処」ではなく「📖 知られていること / 💭 振り返りメモ」と表現

設計の決定事項:
・縦軸 = 体重 (kg)、上端 = 開始体重 (Profile.startingBodyWeightKg)、下端 = 期間に応じて調整

・体重は HealthKit 毎日取得、週/月/3ヶ月で日次の細点 + 週/月の太点を出し分け
・「理論 vs 実測カード」は完全削除、チャート直下の 3 セル数値リードアウト で代替
・痩せ方ペース pill (上部) は環境設定 (Profile) へのショートカット — タップで切替モーダル

実装メモ:
・痩せ方の3パターン (memory 要確認): ゆるやか / ふつう / ハード で kg/週レートを変える設定
・UserGoals.startingBodyWeightKg + 痩せ方レート → 理想ペース直線を計算
・累積カロリー貯金 ÷ 7700 → 理論減量 g、開始体重から引いて kg にして描画
・weightHistoryProvider (HealthKit) を毎日値で取って実測線、移動平均は別途検討
・期間切替で y-axis tick interval を自動調整 (週=100g / 月=250g / 3ヶ月=500g)
・fl_chart の LineChart に 3 系列 + 共通 kg 軸で実装可能
