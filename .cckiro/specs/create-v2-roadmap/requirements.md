# トントン Ver.2.0までのロードマップ - 要件定義（シンプル版）

## 1. プロジェクト概要

### 1.1 アプリケーション概要
- **アプリ名**: トントン (TonTon)
- **コンセプト**: 「カロリー貯金で体重管理をシンプルに」
- **現在バージョン**: v1.0.1
- **目標バージョン**: v2.0
- **目標リリース日**: 2025年12月
- **想定利用期間**: 100日程度

### 1.2 設計思想
「カロリー貯金」というシンプルなメタファーで体重管理を行う。複雑な機能は排除し、誰でも直感的に使える貯金体験に集約する。

## 2. 前提条件

### 2.1 現在の状況
- v1.0.1 リリース済み（App Store提出済み）
- App Store審査対応が進行中
- Flutter iOS アプリとして開発済み
- Supabase バックエンドを使用
- AI食事認識機能実装済み

### 2.2 技術スタック
- **フロントエンド**: Flutter (iOS)
- **バックエンド**: Supabase
- **AI**: GPT-4 Turbo (食事認識)
- **データベース**: PostgreSQL (Supabase)
