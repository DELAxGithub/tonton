# カロリー収支理論の再考と新機能設計

## 1. 背景：カロリー収支論はなぜ破綻しているか

「カロリー収支論（摂取カロリー＜消費カロリーで痩せるという理論）」が破綻している、あるいは通用しないという主張は、近年ダイエットや栄養学の分野で非常によく議論されるテーマです。
結論から言うと、「物理的なエネルギーの法則（熱力学）としては正しいが、人間の生体メカニズムとしては単純すぎる」という見方が強まっています。

### カロリー収支論が通用しないとされる主な理由

1. **「カロリーの質」が無視されている（ホルモンへの影響）**
   - 100kcalのクッキーと100kcalのブロッコリーは、身体にとって同じではない
   - 糖質はインスリン（脂肪を溜め込むホルモン）を分泌させ、脂質やタンパク質とは代謝経路が異なる
   - カロリーだけを制限しても、高糖質な食事を続けていると血糖値が乱高下し、脂肪が燃焼されにくい

2. **代謝の適応（ホメオスタシス）**
   - カロリー制限を続けると、身体は「飢餓状態」と認識し、基礎代謝を減らす
   - 食べる量を減らしても、一定期間で体重が減らなくなる（停滞期）

3. **個人の消化・吸収能力の違い**
   - 食べたカロリーがすべて吸収されるわけではない
   - 食品の形態や腸内細菌叢（マイクロバイオーム）によって実質的な摂取エネルギーは変わる

4. **活動量・カロリー計算の不正確さ**
   - 食品ラベルのカロリー表示は最大20%の誤差が許容されている
   - スマートウォッチの消費カロリーは実際より大幅に過大評価される

5. **空腹感とストレス**
   - 極端な制限はストレスホルモン「コルチゾール」を増やし、食欲を増進させ、脂肪を蓄積しやすくする

### 現在の主流な見解

「カロリー収支」は物理の理論としては正しい（＝脂肪を減らすにはエネルギー赤字が必要）が、「何を食べるか」「身体の代謝をどう整えるか」を無視してカロリーだけを管理するアプローチは失敗する可能性が高い。
血糖値を安定させる食事（低糖質、高タンパク、高食物繊維）やホルモンバランス、食事のタイミングを重視する方が持続可能。

## 2. 現在のtontonの状況

### 持っているデータ
- **PFC（タンパク質・脂質・炭水化物）**: 食事ごとにグラム単位で記録済み
- **カロリー**: 食事ごと + HealthKitから消費カロリー
- **体重・体脂肪率**: HealthKit連携
- **ユーザープロファイル**: 性別・年齢層 → PFC目標値の自動計算あり

### 問題点：データはあるのに活かせていない
- コアUI = 「カロリー貯金」メタファー（カロリー収支だけが主役）
- PFCバーはHomeScreenに表示されるが、**評価・フィードバックがない**
- 「今日のPFCバランスが良いか悪いか」が分からない
- AIアドバイスもカロリーベースの可能性が高い

---

## 3. 設計方針：カロリーから「栄養バランス」へ

### 基本コンセプト
カロリーの数字を追うのではなく、**PFCバランスの質を評価**して、ユーザーに分かりやすくフィードバックする。

### 3.1 食事スコア（ミールスコア）
食事ごとにPFCバランスを評価し、スコアをつける。

- PFC比率が目標に近い → 高スコア
- タンパク質が十分 → ボーナス
- 炭水化物過多 → 減点（インスリンスパイクリスク）
- 例: ⭐⭐⭐⭐☆ (4/5) 「タンパク質◎ 炭水化物やや多め」

### 3.2 1日の栄養バランスダッシュボード
カロリー貯金の代わりに（or 併用して）：

- PFC達成率リング（3つの円グラフ）
- 「今日の栄養バランス」を一言で表示（例: 「タンパク質が足りません」）
- 目標PFCとの差分を視覚的に表示

### 3.3 AIアドバイスの強化
Geminiへのプロンプトを改善：

- カロリーだけでなくPFCバランスを考慮したアドバイス
- 「次の食事で何を補うべきか」の具体的提案
- 「炭水化物が多い日が続いています」等のトレンド分析

### 3.4 週間・月間のPFCトレンド
カロリー貯金グラフに加えて：

- PFCバランスの推移グラフ
- 「タンパク質が安定して目標を達成できている」等の評価
